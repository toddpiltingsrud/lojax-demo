Modals

When querying an AJAX response for new content, lojax also checks incoming content for the presence of a bootstrap modal. If it finds one, it instantiates a bootstrap modal with the new content and displays it.

Treating modals in this way makes them first class citizens along with full-blown pages. This makes your modals re-usable because you can call them from anywhere with a simple button or hyperlink.

Bootstrap does not support multiple open modals. lojax enforces this by checking to see if a modal is already open before opening another one. If there is already a modal on the page, lojax runs the new content through the normal process, checking for the presence of jx-panel attributes.

So here's what you do: place jx-panel attributes on your header, body and footer divs inside your modal layout. If you do, lojax will replace the content of the open modal with the content of the new one.

By the way, this is an excellent opportunity to use the jx-preload attribute.
