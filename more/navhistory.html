Navigation History

lojax uses the URL's hash to track navigation history. To place an item into the browser's navigation history, preface the URL with "#". 

<a href="#navigationhistory.html" data-method="ajax-get">navigation history</a>

lojax handles the "hashchange" event and queries the page for an anchor with a name that matches the hash. If it doesn't find one, it executes an ajax-get using the hash as the URL.

lojax places a restriction on the hashes it will execute. Hashes must at the very least contain two letters followed by a slash, then two more letters (ab/cd). Here's the regular expression lojax uses to decide whether a hash is a URL or not:

/#((.*)?[a-z]{2}\/[a-z]{2}(.*)?)/i

So what happens when the user clicks the back button all the way to the beginning, removing the hash completely?

You can configure what happens in this case by setting lojax.emptyHashAction:

?
1
2
3
$(function () {
    lojax.emptyHashAction = 'home.html';
});

Navigation history tracking is disabled by default. Enable it by setting the navHistory configuration setting to true:

?
1
lojax.config.navHistory = true;
